## 摘要
近年来，多模态大语言模型（MLLMs）在“以图思考”（TwI）范式下，通过迭代执行各种视觉工具操作，在细粒度感知任务上取得了显著进展。然而，该范式严重依赖频繁的外部工具调用和重复的图像重新编码，导致巨大的计算开销和推理延迟。为解决这些问题，我们提出超越视觉（BEE），一种以自调节能力为核心的新型隐式视觉工具范式。BEE直接将视觉工具调用行为纳入训练目标，并鼓励模型发展自调节调用机制。该设计使模型能够自适应地平衡内部知识和隐式工具，避免冗余工具使用，同时大幅降低推理延迟。具体而言，BEE包含两阶段训练过程：（1）形式化思维链（CoT）监督微调（SFT）。我们构建带有结构化工具槽和混合调用状态的CoT轨迹。此阶段激活模型的隐式工具表示和自适应切换能力。（2）自调节奖励驱动对齐。为解决认知边界模糊导致的冗余工具使用问题，我们首先引入净工具增益（NTG）指标来量化该现象。基于此观察，我们进一步提出自调节奖励机制。该机制惩罚无效工具依赖，鼓励模型进行知识路由，确保仅在模型内部知识不足时调用隐式工具。BEE在细粒度视觉感知上达到最先进性能，同时在通用推理任务中保持竞争力，并在推理效率上实现显著提升。

## Abstract
Recent multimodal large language models (MLLMs) have made remarkable progress on fine-grained perception tasks under the "Thinking with Images" (TwI) paradigm by iteratively performing various visual tool operations. However, this paradigm relies heavily on frequent external tool calls and repeated image re-encoding, which leads to substantial computational overhead and inference latency. To address these issues, we propose Beyond the Eye (BEE), a novel implicit visual tool paradigm centered on self-regulated capability. BEE directly incorporates visual tool invocation behaviors into the training objective and encourages the model to develop a self-regulated invocation mechanism. This design enables the model to adaptively balance internal knowledge and implicit tools, avoiding redundant tool usage while substantially reducing inference latency. Specifically, BEE involves a two-stage training process: (1) Formalized Chain-of-Thought (CoT) Supervised Fine-tuning (SFT). We construct CoT trajectories with structured tool slots and mixed invocation states. This stage activates the model's implicit tool representations and adaptive switching capability. (2) Self-regulated Reward-Driven Alignment. To address redundant tool usage caused by ambiguous cognitive boundaries, we first introduce the Net Tool Gain (NTG) metric to quantify this phenomenon. Based on this observation, we further propose a self-regulated reward mechanism. This mechanism penalizes ineffective tool dependency and encourages the model to perform knowledge routing, ensuring that implicit tools are invoked only when the model's internal knowledge is insufficient. BEE achieves state-of-the-art performance in fine-grained visual perception while remaining competitive in general reasoning tasks and achieving substantial gains in inference efficiency.

## 2. 方法论：BEE 框架

论文提出 **Beyond the Eye (BEE)** 框架，其核心思想是将“调用视觉工具”这一行为内化为模型的隐式能力，并通过自我调节机制来决定何时使用。

-   **核心思想**：用**隐式视觉工具 (Implicit Visual Tools, IvT)** 取代外部工具调用。模型在推理时，不再调用外部 API 生成新图像，而是直接生成代表视觉操作结果的语义特征 token（隐式令牌），并将其融入推理链。
-   **关键技术细节**：BEE 包含两阶段训练。

    -   **阶段一：形式化 CoT 监督微调**
        -   **目标**：激活模型的隐式工具表示、自适应切换能力和基础推理能力。
        -   **数据构建**：从 Thyme, Visual-CoT 等数据源生成包含“代码式工具调用”的推理轨迹，然后将代码部分替换为预训练语义分词器生成的隐式令牌（Tool Pad），形成包含隐式工具槽的 CoT 标签。
        -   **混合训练**：数据集同时包含“需要工具”和“无需工具”的推理轨迹，以引导模型学习何时调用工具。
    -   **阶段二：自调节奖励驱动对齐**
        -   **目标**：解决微观微调后的模型过度依赖工具的问题。
        -   **NTG 指标**：引入**“净工具增益” (Net Tool Gain, NTG)** 指标，量化工具调用带来的实际边际收益，揭示了模型存在大量冗余工具调用。
        -   **自调节奖励 (MC-Reward)**：提出一种复合奖励函数 \(R(o_i) = r_{acc} + r_{cons} + r_{fmt} + r_{mc}\)。其中核心是 \(r_{mc}\)，它根据模型在当前 rollout 中的**群体平均准确率 (p)** 动态调整对工具使用的奖惩：
            -   当任务简单（\(p\) 高于阈值 \(\tau\)），使用工具会受到惩罚，鼓励模型依赖自身知识。
            -   当任务困难（\(p\) 低于阈值 \(\tau\)），使用工具会得到奖励，鼓励模型调用隐式工具。
        -   **优化算法**：使用 DAPO 算法进行训练。
    -   **公式原理 (文字说明)**：自调节奖励的核心是一个连续光滑函数 \( \Delta A(p) \)，它决定了使用工具（R_tool）和不使用工具（R_notool）的奖励差。通过设计线性基础奖励和余弦波动的工具奖励项，该函数在 \(p = \tau\) 处形成平滑的转换边界，使模型能根据任务难度和自身能力进行“知识路由”决策。

## 6. 主要结论与发现

1.  **性能卓越**：BEE（特别是 BEE-8B）在细粒度视觉感知基准（如 V* Bench, HRBench, MME-RealWorld-Lite）上达到了**最先进水平**，超越了参数量大得多的 Qwen3-VL-235B、GLM-4.5V 和 Kimi-K2.6 等模型。
2.  **效率显著提升**：与外部工具方法（如 DeepEyes, Thyme）相比，BEE 实现了**1-2个数量级**的推理加速（例如在 8K 分辨率下比 Thyme 快 86.2%），并大幅降低了成本。
3.  **自调节能力有效**：通过 MC-Reward 机制，模型学会了根据任务难度自适应地调用隐式工具，显著减少了冗余工具调用（NTG 从 6.2% 提升至 66.0%），实现了内部知识与外部帮助的优雅平衡。
4.  **泛化能力**：虽然主要针对感知任务，BEE 在通用推理和幻觉检测基准上的表现也优于其基础模型，说明性能提升并非简单的过拟合。

## 7. 优点

-   **方法创新**：
    -   **“隐式化”工具**：将外部工具调用内化，是解决效率瓶颈的根本性创新，避免了 I/O 开销和级联错误。
    -   **“自我调节”机制**：MC-Reward 是一个优雅的设计，它赋予了模型元认知能力（了解自己的能力边界），是实现自适应推理的关键。
-   **实验设计**：
    -   **指标清晰**：提出的 NTG 指标能有效量化工具使用的价值，是评价 Agentic 模型的一个实用工具。
    -   **控制变量严谨**：通过“替换为原始图像”、“替换为无效令牌”等消融实验，确凿证明了隐式工具的有效性。
    -   **评估全面**：覆盖了从感知、推理到实际部署效率的方方面面，论证有力。
    -   **开源贡献**：公开代码、模型和训练数据，可复现性强。

## 8. 不足与局限

-   **复杂推理的局限性**：论文在 Limitations 部分坦诚指出，BEE 在处理需要深度、多步逻辑推理的任务（如数学竞赛题）时，有时会跳过中间步骤，给出直接但不严谨的答案。原因在于训练数据中缺乏这类复杂推理样本。
-   **密集场景的感知局限**：在处理包含大量相似物体的场景（如自动驾驶中多车辆识别）时，BEE 可能会遗漏部分目标。这可能是因为训练数据中缺乏此类密集感知的例子。
-   **未尝试更强的生成式结构**：虽然性能表现优异，BEE 使用的结构（如分词器、注意力机制）相对标准，没有探索更先进的隐式推理结构（如潜在空间中的连续思维链），这可能是未来潜在的增长点。
