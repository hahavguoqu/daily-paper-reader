## 摘要
近年来，多模态大语言模型在“以图思考”范式下，通过迭代执行各种视觉工具操作，在细粒度感知任务上取得了显著进展。然而，该范式严重依赖频繁的外部工具调用和重复的图像重新编码，导致大量计算开销和推理延迟。为解决这些问题，我们提出超越视觉——一种以自我调节能力为核心的新型隐式视觉工具范式。该范式直接将视觉工具调用行为融入训练目标，并鼓励模型发展自我调节的调用机制。这种设计使模型能够自适应地平衡内部知识与隐式工具，避免冗余工具使用，同时大幅降低推理延迟。具体而言，超越视觉涉及两阶段训练过程：（1）形式化思维链监督微调。我们构建带有结构化工具槽和混合调用状态的思维链轨迹。该阶段激活模型的隐式工具表征和自适应切换能力。（2）自我调节的奖励驱动对齐。为解决认知边界模糊导致的冗余工具使用问题，我们首先引入净工具增益指标来量化该现象。基于此观察，我们进一步提出自我调节奖励机制。该机制惩罚无效工具依赖，鼓励模型进行知识路由，确保仅在模型内部知识不足时调用隐式工具。超越视觉在细粒度视觉感知上达到最先进性能，同时在通用推理任务中保持竞争力，并在推理效率上取得显著提升。

## Abstract
Recent multimodal large language models (MLLMs) have made remarkable progress on fine-grained perception tasks under the "Thinking with Images" (TwI) paradigm by iteratively performing various visual tool operations. However, this paradigm relies heavily on frequent external tool calls and repeated image re-encoding, which leads to substantial computational overhead and inference latency. To address these issues, we propose Beyond the Eye (BEE), a novel implicit visual tool paradigm centered on self-regulated capability. BEE directly incorporates visual tool invocation behaviors into the training objective and encourages the model to develop a self-regulated invocation mechanism. This design enables the model to adaptively balance internal knowledge and implicit tools, avoiding redundant tool usage while substantially reducing inference latency. Specifically, BEE involves a two-stage training process: (1) Formalized Chain-of-Thought (CoT) Supervised Fine-tuning (SFT). We construct CoT trajectories with structured tool slots and mixed invocation states. This stage activates the model's implicit tool representations and adaptive switching capability. (2) Self-regulated Reward-Driven Alignment. To address redundant tool usage caused by ambiguous cognitive boundaries, we first introduce the Net Tool Gain (NTG) metric to quantify this phenomenon. Based on this observation, we further propose a self-regulated reward mechanism. This mechanism penalizes ineffective tool dependency and encourages the model to perform knowledge routing, ensuring that implicit tools are invoked only when the model's internal knowledge is insufficient. BEE achieves state-of-the-art performance in fine-grained visual perception while remaining competitive in general reasoning tasks and achieving substantial gains in inference efficiency.

### 2. 方法论：核心思想、关键技术细节

#### 核心思想：BEE - 自调节隐式视觉工具范式

BEE将视觉工具的调用行为**融入模型训练目标**，使其成为模型内部的“隐式工具”。核心是让模型学习一个**自我调节的调用机制**，基于对自身能力的认知（能力边界awareness）来决定何时使用隐式工具。

#### 关键技术细节：两阶段训练

- **第一阶段：形式化思维链监督微调**
    - **目标**：激活模型使用隐式工具的表征能力，并学习基本的自适应切换机制。
    - **数据构建**：
        1.  从 **Thyme**、**Refocus** 等数据源生成带有显式工具调用（如代码）的推理轨迹。
        2.  使用**语义工具分词器**，将显式工具操作生成的辅助图像编码为离散的语义特征Token，替换代码段。
        3.  构建包含**混合调用状态**的CoT轨迹：部分样本包含隐式工具槽（需要工具），部分不包含（不需要工具），以监督模型学习何时调用工具。
    - **数据集**：BEE-SFT-320K，包含约32万条样本，覆盖缩放、裁剪、旋转等6种视觉操作。

- **第二阶段：自我调节的奖励驱动对齐**
    - **目标**：引导模型建立清晰的能力边界，抑制无效冗余的工具调用。
    - **关键概念：净工具增益（NTG）**：一个新的量化指标，用于衡量工具调用带来的实际边际收益，排除模型基础能力的影响。NTG = (有效工具收益 - 工具引入幻觉) / 工具调用总数。分析发现，仅经过SFT的模型NTG很低（6.2%），存在大量冗余调用。
    - **核心机制：自我调节奖励（MC-Reward）**：
        - **原理**：在训练rollout阶段，根据模型在某一问题上的**群体平均正确率**（p）动态判断问题难度。
        - **规则**：
            - 当`p`低于阈值`τ`时，认为是困难问题，奖励函数**鼓励**使用隐式工具。
            - 当`p`高于或等于`τ`时，认为是简单问题，模型应依靠自身能力解决，奖励函数**惩罚**不必要的工具调用。
        - **实现**：奖励函数由两部分组成：一个与能力指标`p`线性相关的基线奖励，和一个专为工具调用设计的余弦门控项。这种平滑设计确保了梯度稳定性。
    - **优化算法**：使用 **DAPO** 算法进行强化学习训练。

- **结论**：实验设计全面，对比充分，结果可信度较高，验证了所提出方法的有效性。

### 6. 主要结论与发现

- **性能领先**：BEE在多个细粒度视觉感知基准上达到**最先进的性能**，甚至超过了参数量大得多的开源模型（如Qwen3-VL-235B、GLM-4.5V）和闭源模型（如Kimi-K2.6）。
- **效率大幅提升**：与外部工具方法（如Thyme、DeepEyes）相比，BEE的推理速度**提升了数倍**（如BEE-7B吞吐量是DeepEyes的23倍），在8K高分辨率下实现了**86.2%的推理加速**。
- **自调节能力有效**：MC-Reward机制成功抑制了冗余的工具调用。训练过程中，模型在简单问题上的工具调用率从近乎100%下降至接近0%，而在困难问题上则保持在约41%的水平。NTG值从6.2%显著提升至66.0%。
- **隐式工具优于文本推理**：消融实验证明，隐式工具Token提供了超出额外文本推理时间带来的增益，对模型性能有实质性贡献。

### 7. 优点

- **方法创新性**：首次提出“自调节隐式视觉工具”范式，通过模型内化工具行为，从根本上解决了外部工具调用的效率和开销问题。
- **设计简洁有效**：通过一个结构清晰的Net Tool Gain指标和巧妙的MC-Reward奖励函数，优雅地实现了模型能力边界与工具调用的对齐。
- **卓越的效率-性能平衡**：BEE在几乎所有测试任务上都优于现有外部工具方法，同时推理延迟显著降低，实现了“又快又好”的目标。
- **高可部署性**：由于移除了对外部环境的依赖，BEE可以像标准大语言模型一样直接部署，而无需额外的沙箱环境，部署成本和门槛低。

### 8. 不足与局限

- **复杂逻辑推理能力不足**：在需要多步、严谨逻辑推理的任务（如VisuLogic， WeMath）上，BEE虽然优于基座模型，但表现仍有较大提升空间。模型有时会跳过推理步骤直接得出答案。
- **密集多物体场景的遗漏**：在存在大量相似物体的场景（如自动驾驶街景）中，BEE有时会遗漏部分目标物体，这可能源于训练数据中此类场景的缺乏。
- **训练数据的依赖与偏差**：模型性能很大程度上依赖于精心构建的SFT数据，该数据混合了带工具和不带工具的轨迹。若数据分布存在偏差（如某些工具使用场景覆盖不足），可能导致模型在某些领域泛化能力受限。
- **对基座模型的依赖**：BEE的最终性能上限仍然受限于其基座模型（Qwen-VL系列）。在通用知识和某些推理能力上，与顶级闭源模型相比仍有差距。
