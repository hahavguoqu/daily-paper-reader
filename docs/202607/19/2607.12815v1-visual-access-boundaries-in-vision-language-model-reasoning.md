## 摘要
思维链提示被广泛用作视觉-语言模型的测试时扩展策略，但仍不清楚当视觉-语言模型生成更长的推理轨迹时，具体扩展了什么。我们探究思维链是否需要持续访问图像标记，还是主要操作于前向传递中已经可用的视觉信息。我们提出视觉访问扫描，一种因果干预方法，沿层深度和生成时间屏蔽从生成标记查询到图像标记键的注意力，并将视觉访问边界定义为保持任务准确性的最小访问区域。在Qwen2.5-VL和InternVL3的六种模型配置中，无思维链直接回答和思维链提示都表现出有限的视觉访问边界。在Qwen2.5-VL-32B以及14B和38B规模的InternVL3中，当思维链与无思维链完全访问目标进行比较时，其视觉访问边界层与无思维链边界的差异最多不超过两层，尽管生成了更长的序列。这表明思维链并非主要通过延长整个推理轨迹中直接图像标记访问来提升性能，而是通过扩展语言端对图像派生隐藏状态信息的计算。我们进一步表明思维链的收益受限于感知读出。当查询的视觉属性能够被模型可靠地读出时，思维链有帮助，但当读出不可靠时则无效。符号属性或acles显示，一旦将真实属性作为文本提供，思维链能改善计数，而单对象探测-解码检查表明，困难属性可以从隐藏状态中线性恢复，但模型本身难以输出。这些分析共同将瓶颈置于读出而非计数。

## Abstract
Chain-of-Thought (CoT) prompting is widely used as a test-time scaling strategy for Vision-Language Models (VLMs), but it remains unclear what is extended when VLMs generate longer reasoning traces. We ask whether CoT requires continued access to image tokens, or whether it mainly operates over visual information already made available earlier in the forward pass. We introduce Visual Access Sweep, a causal intervention that masks attention from generated-token queries to image-token keys along layer depth and generation time, and define the Visual Access Boundary (VAB) as the minimal access region that preserves task accuracy. Across six model configurations from Qwen2.5-VL and InternVL3, both no-CoT direct answering and CoT prompting exhibit finite VABs. In Qwen2.5-VL-32B and InternVL3 at 14B and 38B scales, when CoT is evaluated against the no-CoT full-access target, its VAB layer differs from the no-CoT boundary by at most two layers, despite substantially longer generations. This suggests that CoT does not primarily improve performance by prolonging direct image-token access throughout the reasoning trace, but by extending language-side computation over image-derived hidden-state information. We further show that CoT gains are constrained by perceptual readout. CoT helps when the queried visual attribute can be reliably read out by the model, but not when that readout is unreliable. A symbolic-attribute oracle shows that CoT can improve counting once ground-truth attributes are supplied as text, while a single-object probe-vs-decode check shows that hard attributes can be linearly recoverable from hidden states yet difficult for the model itself to output. Together, these analyses place the bottleneck at readout rather than counting.

## 1. 核心问题与整体含义（研究动机和背景）

思维链（Chain-of-Thought, CoT）提示被广泛用作视觉-语言模型（Vision-Language Model, VLM）的测试时扩展策略，但其作用机制尚不明确。现有研究大多通过观察注意力衰减来推测，但注意力大小无法区分“持续访问不再必要”和“访问失败”。因此，作者提出一个因果性问题：CoT 是通过延长对图像 token 的直接访问来提升性能，还是主要借助早期已获取的视觉信息进行额外语言侧计算？澄清这一问题对理解 VLM 推理机制、优化测试时扩展策略具有重要意义。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
作者提出 **Visual Access Sweep**（视觉访问扫描），一种因果干预方法：沿**层深度**和**生成时间**两个维度，屏蔽从生成 token 查询（query）到图像 token 键（key）的直接注意力，从而测试哪些区域的直接图像 token 访问对任务精度是**功能上必要的**。

### 关键技术细节
- **操作定义**：在标准自注意力中，对注意力掩码 \( M_\ell \) 设置 \( M_{\ell ij} = -\infty \) 对应被屏蔽的 query–key 对，使 softmax 后权重为 0。
- **扫描参数**：
  - 层截止 \( \ell_{\text{cutoff}} \)：前 \( \ell_{\text{cutoff}} \) 层允许访问，之后屏蔽。
  - token 截止 \( t_{\text{cutoff}} \)：前 \( t_{\text{cutoff}} \) 个生成 token 允许访问，之后屏蔽。
  - 有效窗口 \( t_{\text{eff}} = \min(t_{\text{cutoff}}, T_{\text{gen}}) \)。
- **Visual Access Boundary (VAB)**：定义为在精度下降不超过容忍度 \( \epsilon \)（文中取 0.05）下的最小访问区域 \( (\ell^*, \tau^*) \)。
- **三种精度下降度量**：
  - \( \Delta_D(\ell,\tau) \)：直接回答（Direct）的下降。
  - \( \Delta_{\text{CoT}}(\ell,\tau) \)：CoT 相对于自身全访问精度的下降（CoT-own-max）。
  - \( \Delta_{DA}(\ell,\tau) \)：CoT 相对于 Direct 全访问精度的下降（direct-anchored）。
- **控制实验**：Null-sink 控制（将移除注意力重定向到虚拟槽）和 late-layer query-text block 控制（额外屏蔽 prompt token→图像 token 注意力），均表明 VAB 结构稳定。

## 6. 主要结论与发现

1. **直接回答和 CoT 都存在有限 VAB**：在所有测试配置中，屏蔽后面层和后面生成 token 的视觉访问不会显著降低精度，只有干预到较早层时才会急剧下降。
2. **CoT 不依赖延长图像 token 访问**：CoT 生成的 token 数比 Direct 多约 50 倍，但 VAB 的层位置差异最多 ±9 层（CoT-own-max 标准），且直接锚定标准下大模型（Qwen2.5-VL-32B、InternVL3-14B/38B）的 CoT VAB 与 Direct 边界相差 ≤ 2 层。表明 CoT 主要通过语言侧计算利用早期已提取的隐藏状态信息。
3. **CoT 收益受限于感知读出能力**：属性级 CoT 增益与多对象感知读出精度正相关（在 Qwen2.5-VL 三个尺度上，Pearson r 为 0.51~0.92，Spearman ρ 为 0.5~1.0）。当读出不可靠（如角度、大小）时，CoT 无明显改进。
4. **瓶颈在读出而非计数运算**：符号属性 Oracle bypass（提供真实属性文本）统一恢复 CoT 增益；线性探测显示困难属性可从隐藏状态中恢复（探测精度高），但模型自身无法可靠解码（低解码精度）。因此，限制因素是模型将视觉信息转化为可用符号的能力，而非计数推理本身。

## 7. 优点

- **因果干预方法**：提出 Visual Access Sweep，从观察注意力转向直接测试必要性，避免了注意力相关性带来的歧义。
- **跨架构、跨规模验证**：在 Qwen2.5-VL（3B/7B/32B）和 InternVL3（8B/14B/38B）共6个配置上一致观察到有限 VAB，增强了结论泛化性。
- **巧妙的任务设计**：通过固定计数操作、变化感知属性，隔离了感知难度和推理复杂性的混淆。
- **多层诊断手段**：结合感知读出精度、线性探测、Oracle bypass 和选择性路由，系统定位了限制因素在读出环节。
- **控制实验严谨**：Null-sink 控制和 query-text block 控制排除了注意力重分配和查询介导重读的替代解释，增加了 VAB 测量的可信度。
- **扩展到真实图像和交叉注意力架构**：GQA 实验和 Llama-3.2 Vision 试点表明 VAB 结构不限于合成场景或前缀融合模型。

## 8. 不足与局限

### 实验覆盖限制
- **主要基于合成计数任务**：属性个数少、场景简单，可能忽略了需要持续迭代视觉检查的复杂任务（如空间关系推理、文本阅读）。作者承认这一点。
- **GQA 实验规模小**：仅使用 100 张图，且 CoT 精度低于 Direct，可能无法有力泛化。
- **未覆盖显式重感知方法**：如 MemVR、Look-Back 等设计上主动重新访问图像的模型，VAB 是否仍有限存疑。
- **角度属性作为限制案例**：全访问精度接近地板，VAB 无法明确定义，削弱了在该属性上的结论。

### 方法与内在机制
- **未揭示 VAB 的成因**：VAB 只表明某区域直接访问不必要，但未能区分是“早期已绑定到隐藏状态”“残差流传递”还是“CoT 前置视觉提取”。需要进一步干预残差流。
- **容忍度 \(\epsilon\) 的影响**：虽然进行了敏感性分析，但 \(\epsilon=0.05\) 的选择仍带有一定主观性，且边界定义基于离散网格，可能引入微小误差。

### 偏差风险
- **模型特定偏差**：仅在四个模型家族（Qwen2.5-VL、InternVL3、Llama-3.2）上测试，未涵盖更多 VLM（如 LLaVA、BLIP-3）。
- **任务特定偏差**：计数操作本身较简单，CoT 的真实收益可能在其他任务（如多步逻辑推理、视觉常识）中有不同表现。
- **Oracle 实验的局限性**：仅测试了 3B 模型的 Oracle 绕过，更大模型上符号信息是否同样恢复 CoT 增益未验证；同时 Oracle 实验无图像输入，无法确保统计分布完全匹配图像场景。

### 应用限制
- 文中选择性路由启发式（阈值 0.5）仅用于说明，未在真实数据集上优化或验证其通用性。
- 实际部署中，对每个属性进行读出检测可能带来额外开销，且存在误差累积风险。
